CMD Agent · integration plan / placeholder

What this page is for

This is the seat for the CMD agent (the autonomous shell/desktop operator that lives in cmd/guiagent/ on the server). It is not a bash REPL — Swarm's job tree and the CMD agent are different processes. When CMD's HTTP surface lands, this page is where its live activity will plug in.

Where CMD will plug in

  • Live screen + thoughts → SSE feed at http://10.0.0.58:5005/stream (already wired into the GUI page; will mirror here with the agent's reasoning trace).
  • Latest screenshot → GET http://10.0.0.58:5005/screenshot.
  • Goal submit → POST http://10.0.0.58:5005/goal (planned) — fires off a CMD task and returns a task_id we can stream below.
  • Task tree → CMD emits step / tool / observation events that will render as a family-tree similar to Swarm's job tree.

Typical CMD plan

  1. Goal received — natural-language request (e.g. "open Brave, search docs, copy the URL").
  2. Plan — CMD breaks it into actionable steps (open app, locate UI element, type, click, verify).
  3. Per-step ReAct loop — screenshot → reason → tool call (mouse/keyboard/shell) → observe → repeat.
  4. Verification — success criteria checked from the next screenshot or shell stdout.
  5. Result — final artifact (URL, file path, exit code) returned to the originating job/user.
